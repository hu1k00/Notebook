Path traversal

  • What is path traversal?

    • Path traversal is also known as directory traversal. These vulnerabilities enable an attacker to read arbitrary files on the server that is running an application. This might include:

      1. Application code and data.
      2. Credentials for back-end systems.
      3. Sensitive operating system files.

    • In some cases, an attacker might be able to write to arbitrary files on the server, allowing them to modify application data or behavior, and ultimately take full control of the server.

  • Reading arbitrary files via path traversal

    • Imagine a shopping application that displays images of items for sale. This might load an image using the following HTML: <img src="/loadImage?filename=218.png">

    • The loadImage URL takes a filename parameter and returns the contents of the specified file. The image files are stored on disk in the location /var/www/images/

    • This application implements no defenses against path traversal attacks. As a result, an attacker can request the following URL to retrieve the /etc/passwd file from the server's filesystem:

      https://insecure-website.com/loadImage?filename=../../../etc/passwd

 - The following is an example of an equivalent attack against a Windows-based server:

 ```https://insecure-website.com/loadImage?filename=..\..\..\windows\win.ini```

  • How to prevent a path traversal attack?

    • Preventing path traversal attacks involves implementing various security measures in your application to ensure that user input cannot be used to access files or directories outside of the intended scope. Here are some best practices to prevent path traversal attacks:

      1. Input Validation: Validate and sanitize all user input, especially file and directory paths, to ensure they contain only allowed characters and do not contain any sequences that could be used to traverse directories.

      2. Use Whitelists: Instead of blacklisting specific characters or patterns, use whitelists to specify the allowed characters and patterns for file and directory names.

      3. File Path Canonicalization: Normalize and canonicalize file paths before using them. This involves converting relative paths to absolute paths and removing any redundant elements such as ".." or ".".

      4. Restrict Access: Set appropriate file system permissions to restrict access to sensitive files and directories. Ensure that files that should not be accessed directly by users are not accessible via URL manipulation.

      5. Use Framework Libraries: Utilize built-in functions or libraries provided by your programming language or framework to handle file operations securely. These libraries often have built-in protections against path traversal attacks.

      6. Implement Access Controls: Implement access controls to restrict the files and directories that users can access based on their privileges and roles. This can help prevent unauthorized access to sensitive files.

      7. Use Sandboxing: Run your application in a sandbox environment with restricted privileges to limit the potential impact of a successful path traversal attack.

      8. Regular Security Audits: Conduct regular security audits of your application's codebase to identify and fix any potential vulnerabilities, including those related to path traversal attacks.

      9. Education and Training: Educate developers about the risks of path traversal attacks and provide training on secure coding practices to help prevent such vulnerabilities from being introduced in the first place.
