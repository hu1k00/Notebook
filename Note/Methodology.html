Methodology

Information Gathering (Recon)

  • Subdomain
    • Subdomain enumeration (Sublist3r)
    • Sublert for domain monitoring
    • Cer.sh [%.Domain.com]
    • gitGraber to search for sensitive information in Github repositories
    • Shodan.io
    • Internet Archive (aka way back machine)
    • Browser plugins: Wappalyzer, and FlagFox
    • LazyRecon
  • Technology

Scanning

  • Port scanning Nmap/ Masscan and BurpScanner

    • Nmap with -A option, and -auth-http-enum scripts (Masscan?)

    • S3Scanner for insecure AWS buckets

    • Subdomain TakeOver: https://github.com/m4110k/takeover && https://github.com/haccer/subjack

    • Run Burp scanner in the background with following plugins: J2EEScan, NGINX Alias Traversal, Telewreck, ParamMiner, and Upload Scanner

  • File and dierctories BruteForcing,in order to identify backup files, test and development files;

    • Dirsearch for backup files, hidden files, and admin interfaces

    • Configure XSS Hunter with Burp Proxy (UserAgent and/or X- Forwarded-For)

  • Test for privilege esclations, and cluent/server side vulnerabilities.

Exploitation

  • Never Access sensitive data , but only retrieve data sample that could be used as {POC}

  • Avoid abusing adminstrative privileges.

Reporting

  • Detailed information about the identified vulnerability

  • How an attacker could wxploit it ? Are certain privileges required? Or any inernet user can exploit it ?
